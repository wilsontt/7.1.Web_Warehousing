# 專案標準文件 (Project Standards)

本目錄存放專案的核心規範文件，這些文件定義了整個專案開發的基礎原則和標準。

## 文件清單

1. **[專案憲章](./1.專案憲章%20Project%20Constitution：文件倉儲管理系統.md)**
   - 定義專案的核心原則和不可協商的要求
   - 包含技術堆疊、安全要求、開發流程等基本規範

2. **[產品需求文件 (PRD)](./PRD-產品需求.md)**
   - 定義產品的整體需求和使用者故事
   - 包含背景、目標、使用者群組、使用場景、功能需求等

3. **[設計規範](./2.文件倉儲管理系統%20Warehouse%20Management%20System%20設計規範v2.0.md)**
   - 定義系統的 UI/UX 設計標準
   - 包含元件規範、佈局標準、互動設計等

4. **[規格驅動開發 (SDD) 工作流程](./3.規格驅動開發%20(SDD)%20工作流程.md)**
   - 定義專案的開發流程和方法論
   - 包含 SPEC、PLAN、TASKS、EXECUTE、REVIEW 五個階段

## 使用方式

所有開發工作都應遵循這些標準文件中的規範。在開始任何模組開發前，請先閱讀相關的標準文件。
